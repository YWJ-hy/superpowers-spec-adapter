description: "Structurally import a user-provided wiki page or directory into project or shared wiki without overwriting content"

Generated by superpower-adapter.

Use this command when the user provides an existing wiki page or directory that should be copied into this adapter's `.superpowers/wiki` or `.shared-superpowers/wiki` tree.

The goal is structural migration: preserve source content, place it under the selected wiki root, avoid overwriting existing different content, and rebuild indexes. This command does not semantically merge imported rules into project-owned or shared-owned wiki pages.

---

## Non-negotiable rules

- Treat the user input as a source wiki page or source wiki directory path.
- Treat this as a standalone adapter command, not as a Superpowers development workflow step.
- Do not invoke Superpowers planning, implementation, review, completion, or verification commands/skills before or after this command.
- Do not invoke `superpowers:verification-before-completion` or similar completion checks for this command.
- Do not discard, summarize away, or silently omit user-provided wiki content.
- Preserve imported source file content as-is.
- Do not overwrite existing wiki files with different content.
- Prefer explicit `--target` when the user wants a specific destination.
- Use `--wiki-root shared` only when the user asks to import into shared wiki or the imported content is clearly cross-project shared knowledge.
- After importing, refresh the selected wiki root's index chain.

## User Input

```text
$ARGUMENTS
```

The user input should identify the source wiki page or directory path. It may also include an explicit target path, target directory, or destination root.

## Recommended import paths

When the user provides a source wiki directory and does not specify shared wiki, default to project wiki:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/wiki_import.py path/to/original-wiki-dir
```

To import into shared wiki:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/wiki_import.py path/to/original-wiki-dir --wiki-root shared
```

The script will:

1. recursively scan supported source files: `.md`, `.markdown`, `.mdx`, `.txt`
2. copy each source file into the selected wiki root using the source relative path
3. skip identical existing files when merge mode is enabled
4. refuse to overwrite existing files with different content
5. rebuild the selected wiki root's indexes

If all source wiki pages should be placed under a target adapter directory:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/wiki_import.py path/to/original-wiki-dir --target imported
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/wiki_import.py path/to/original-wiki-dir --wiki-root shared --target imported
```

If a single source file should go to a specific adapter leaf:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/wiki_import.py path/to/source-wiki-page.md --target imported/source-wiki-page.md
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/wiki_import.py path/to/source-wiki-page.md --wiki-root shared --target imported/source-wiki-page.md
```

Use `--merge-existing` only to allow identical existing files to be skipped. It never overwrites different content.

## About `--hint`

The import script may accept `--hint` for compatibility with older adapter flows, but structural import does not use it to choose a semantic target. If imported content needs semantic consolidation, use the `update-wiki` skill after import.

## Existing wiki merge behavior

If the selected wiki root already has files, import must preserve them.

The script does this by:

- using the explicit `--target` when provided, otherwise preserving source-relative paths under the selected wiki root
- copying source content as-is
- skipping identical files when `--merge-existing` is set
- refusing to overwrite different existing files
- rebuilding indexes after import

## Relationship to update-wiki

Use `import-wiki` for one-time structural migration of existing user wiki pages into `.superpowers/wiki/` or `.shared-superpowers/wiki/`.
Use the `update-wiki` skill later to semantically consolidate, deduplicate, or maintain durable implementation knowledge discovered during implementation, debugging, review, or discussion.

Generated by superpower-adapter.

Use this command as a manual diagnostic when automatic Superpowers plan sidecar preparation is blocked, ambiguous, or needs explanation.

Daily workflow preparation is normally automatic: the SessionStart hook attempts planning sidecar preparation for the current plan, and the `plan-context-sidecar` skill runs implement/review gates before using selected context. This command remains available to inspect and repair state explicitly.

It helps you diagnose that:

- the target plan is valid
- the sidecar context is initialized automatically
- planning-selected context is selected and written before implementation
- review has enough context to proceed safely
- completion considers whether durable knowledge should be written back to `.superpowers/spec/`

---

## When to Use

Do not require users to run this in the normal path. Use it when:

- `.superpowers/current-plan` may be stale or missing
- automatic SessionStart sidecar preparation did not write `plan.jsonl`
- implement or review gate returned `BLOCK`
- the task should reuse planning-selected context but the sidecar seems incomplete
- you suspect the sidecar may be incomplete or invalid
- you need an explicit completion check before deciding whether to run `/update-spec`

## Commands

### Planning

If you know the target plan:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py planning --plan docs/superpowers/plans/<stem>.md --hint "$ARGUMENTS"
```

Use this manually only when SessionStart did not prepare the current plan, or when you need to target an explicit plan. The planning gate initializes the plan sidecar, selects relevant indexed specs from `.superpowers/spec/`, and writes planning context into `plan.jsonl` automatically.

Expected outcomes:

- `OK`: the plan, sidecar, and planning-selected context are ready
- `WARN`: the plan exists but automatic spec selection found no strong candidate, or inputs are incomplete
- `BLOCK`: the plan path is invalid or the plan file cannot be resolved

If automatic selection returns `WARN`, refine the plan summary or rerun planning with a more specific `--hint`.

### Implement

For manual diagnosis before implementation:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py implement
```

Or target an explicit plan:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py implement --plan docs/superpowers/plans/<stem>.md
```

This checks that:

- the current or explicit plan resolves
- the sidecar exists
- `plan.jsonl` contains planning-selected spec context
- sidecar state is structurally valid

If implementation is blocked because `plan.jsonl` is empty, do not start coding yet. Rerun the planning gate with a better hint so it can automatically select and write planning context:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py planning --plan docs/superpowers/plans/<stem>.md --hint "<task keywords>"
```

### Completion

For explicit diagnostics before finishing a task:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py completion --summary "<task summary>"
```

Optionally include changed files:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py completion \
  --summary "<task summary>" \
  --changed-file src/foo.ts \
  --changed-file src/bar.ts
```

This checks that:

- the current workflow state is still coherent
- completion has enough input to judge whether spec updates should be considered
- the task may have produced durable implementation knowledge

If completion returns a durable-knowledge warning, consider updating `.superpowers/spec/`:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/spec_update_run.py "error handling" "Error normalization" "Prevent inconsistent API error shapes." "Normalize API error payloads"
```

## Output Semantics

### OK

You can proceed.

### WARN

You may proceed, but something is incomplete, weak, or missing.
Read the warning and decide whether to repair the workflow state first.

Typical `WARN` cases:

- sidecar not initialized during planning
- `review.jsonl` has no records yet
- completion lacks enough summary information
- the task may have produced durable knowledge worth writing back to spec

### BLOCK

Do not proceed yet.
A required precondition is missing or invalid.

Typical `BLOCK` cases:

- no current plan and no explicit `--plan`
- the plan path is invalid
- sidecar state is broken
- `plan.jsonl` is missing or empty before implementation
- JSONL or `state.json` is malformed

## Recommended Handling Rules

### If planning returns `WARN`

Usually rerun planning with a more specific hint so the gate can select specs automatically:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py planning --plan docs/superpowers/plans/<stem>.md --hint "<task keywords>"
```

### If implement returns `BLOCK`

Do not start implementation.
Fix the workflow state first:

- resolve the correct plan
- initialize the sidecar if needed
- add planning-selected spec context to `plan.jsonl`

### If review returns `WARN`

Decide whether planning-selected context is enough.
If not, add review-specific context before reviewing.

### If completion returns `WARN`

Consider whether the task changed:

- contracts
- validation behavior
- implementation conventions
- non-obvious gotchas
- reusable patterns

If yes, run the `update-spec` workflow.

## Minimal Safe Loop

The normal loop is mostly automatic:

1. Superpowers creates or selects a plan and `.superpowers/current-plan` points to it.
2. SessionStart attempts planning sidecar preparation and writes planning context.
3. `plan-context-sidecar` runs the implement gate and renders implementation context.
4. Implement with rendered context when needed.
5. `plan-context-sidecar` runs the review gate and renders review context.
6. Review with rendered context when needed.
7. Update spec if the task produced durable knowledge.

Manual diagnostic example when automatic preparation is blocked:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py planning --plan docs/superpowers/plans/<stem>.md --hint "<task keywords>"
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py implement --json
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/plan-context.py render --phase implement
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py review --json
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/plan-context.py render --phase review
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/workflow-gate.py completion --summary "<task summary>"
```

Generated by superpower-adapter.

Use this command before entering a new workflow phase when the project uses Superpowers plans and plan sidecar context.

This command helps you verify that:

- the target plan is valid
- the sidecar context is initialized
- planning-selected context exists before implementation
- review has enough context to proceed safely
- completion considers whether durable knowledge should be written back to `.superpowers/spec/`

---

## When to Use

Run this before:

- planning from a plan file
- starting implementation
- starting review
- wrapping up a task

Use it especially when:

- `.superpowers/current-plan` may be stale or missing
- the task depends on `.superpowers/spec/`
- the task should reuse planning-selected context instead of rediscovering specs
- you suspect the sidecar may be incomplete or invalid

---

## Commands

### Planning

If you know the target plan:

```bash
python3 superpowers/scripts/workflow-gate.py planning --plan docs/superpowers/plans/<stem>.md
```

Use this before selecting specs for a new or existing plan.

Expected outcomes:

- `OK`: the plan and sidecar are ready
- `WARN`: the plan exists but the sidecar is missing or incomplete
- `BLOCK`: the plan path is invalid or the plan file cannot be resolved

If the sidecar is missing, initialize it:

```bash
python3 superpowers/scripts/plan-context.py init docs/superpowers/plans/<stem>.md --set-current
```

---

### Implement

Before implementation:

```bash
python3 superpowers/scripts/workflow-gate.py implement
```

Or target an explicit plan:

```bash
python3 superpowers/scripts/workflow-gate.py implement --plan docs/superpowers/plans/<stem>.md
```

This checks that:

- the current or explicit plan resolves
- the sidecar exists
- `plan.jsonl` contains planning-selected spec context
- sidecar state is structurally valid

If implementation is blocked because `plan.jsonl` is empty, do not start coding yet. First:

1. read `.superpowers/spec/index.md`
2. follow the index chain
3. select the minimum relevant spec files
4. add them to the plan phase context

Example:

```bash
python3 superpowers/scripts/plan-context.py add --phase plan --spec .superpowers/spec/topic/example.md --reason "Needed for implementation safety"
```

---

### Review

Before review:

```bash
python3 superpowers/scripts/workflow-gate.py review
```

Or target an explicit plan:

```bash
python3 superpowers/scripts/workflow-gate.py review --plan docs/superpowers/plans/<stem>.md
```

This checks that:

- the plan resolves
- the sidecar is valid
- planning-selected context exists
- review context exists or is intentionally absent

Review may return `WARN` if `review.jsonl` is empty. In that case, decide whether the planning-selected context is already sufficient. If not, add review-specific context first:

```bash
python3 superpowers/scripts/plan-context.py add --phase review --spec .superpowers/spec/topic/example.md --reason "Needed for review criteria"
```

---

### Completion

Before finishing a task:

```bash
python3 superpowers/scripts/workflow-gate.py completion --summary "<task summary>"
```

Optionally include changed files:

```bash
python3 superpowers/scripts/workflow-gate.py completion \
  --summary "<task summary>" \
  --changed-file src/foo.ts \
  --changed-file src/bar.ts
```

This checks that:

- the current workflow state is still coherent
- completion has enough input to judge whether spec updates should be considered
- the task may have produced durable implementation knowledge

If completion returns a durable-knowledge warning, consider updating `.superpowers/spec/`:

```bash
python3 superpowers/scripts/spec_update_run.py "error handling" "Error normalization" "Prevent inconsistent API error shapes." "Normalize API error payloads"
```

## Output Semantics

### OK

You can proceed.

### WARN

You may proceed, but something is incomplete, weak, or missing.
Read the warning and decide whether to repair the workflow state first.

Typical `WARN` cases:

- sidecar not initialized during planning
- `review.jsonl` has no records yet
- completion lacks enough summary information
- the task may have produced durable knowledge worth writing back to spec

### BLOCK

Do not proceed yet.
A required precondition is missing or invalid.

Typical `BLOCK` cases:

- no current plan and no explicit `--plan`
- the plan path is invalid
- sidecar state is broken
- `plan.jsonl` is missing or empty before implementation
- JSONL or `state.json` is malformed

## Recommended Handling Rules

### If planning returns `WARN`

Usually initialize the sidecar before proceeding:

```bash
python3 superpowers/scripts/plan-context.py init docs/superpowers/plans/<stem>.md --set-current
```

### If implement returns `BLOCK`

Do not start implementation.
Fix the workflow state first:

- resolve the correct plan
- initialize the sidecar if needed
- add planning-selected spec context to `plan.jsonl`

### If review returns `WARN`

Decide whether planning-selected context is enough.
If not, add review-specific context before reviewing.

### If completion returns `WARN`

Consider whether the task changed:

- contracts
- validation behavior
- implementation conventions
- non-obvious gotchas
- reusable patterns

If yes, run the `update-spec` workflow.

## Relationship to Other Adapter Commands

- `plan-context.py init` initializes sidecar state
- `plan-context.py add` records selected spec context for a phase
- `plan-context.py render` materializes selected context for implementation or review
- `spec_update_run.py` writes durable knowledge back into `.superpowers/spec/`

Use `check-workflow` to decide whether the workflow is ready before using those commands.

---

## Minimal Safe Loop

A typical loop looks like this:

1. Check planning readiness
2. Initialize sidecar if needed
3. Select and record planning context
4. Check implementation readiness
5. Implement with rendered context
6. Check review readiness
7. Review with rendered context
8. Check completion readiness
9. Update spec if the task produced durable knowledge

Example:

```bash
python3 superpowers/scripts/workflow-gate.py planning --plan docs/superpowers/plans/<stem>.md
python3 superpowers/scripts/plan-context.py init docs/superpowers/plans/<stem>.md --set-current
python3 superpowers/scripts/plan-context.py add --phase plan --spec .superpowers/spec/topic/example.md --reason "Required for planning"
python3 superpowers/scripts/workflow-gate.py implement
python3 superpowers/scripts/plan-context.py render --phase implement
python3 superpowers/scripts/workflow-gate.py review
python3 superpowers/scripts/plan-context.py render --phase review
python3 superpowers/scripts/workflow-gate.py completion --summary "<task summary>"
```

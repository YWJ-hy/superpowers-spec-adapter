---
description: "Convert a user-provided spec directory into adapter .superpowers/spec format without losing source content"
---

Generated by superpower-adapter.

Use this command when the user provides an existing spec directory whose files, format, or style do not match this adapter's `.superpowers/spec` conventions.

The goal is to scan all source spec files, route them into adapter-style leaf specs under `.superpowers/spec/`, merge into existing adapter specs when present, and preserve every source file verbatim.

---

## Non-negotiable rules

- Treat the user input as a source spec file or source spec directory path.
- Treat this as a standalone adapter command, not as a Superpowers development workflow step.
- Do not invoke Superpowers planning, implementation, review, completion, or verification commands/skills before or after this command.
- Do not invoke `superpowers:verification-before-completion` or similar completion checks for this command.
- Do not discard, summarize away, or silently omit user-provided spec content.
- Preserve every imported source file inside the converted target file.
- Merge into existing `.superpowers/spec` files by appending imported blocks; do not overwrite existing adapter spec content.
- Keep converted adapter-facing sections structured as `## Update:` blocks with `Why`, `Rules / Contracts`, and `Validation / Notes`.
- Prefer leaf spec targets, not `index.md`, unless the user explicitly asks for an index-level import.
- If the source contains ambiguous or conflicting content, preserve it and mark it for review instead of deleting it.
- After importing, refresh the spec index chain.

## User Input

```text
$ARGUMENTS
```

The user input should identify the source spec directory path. It may also include an optional hint or target.

## Recommended directory import path

When the user provides a source spec directory:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/spec_import.py path/to/original-spec-dir
```

The script will:

1. recursively scan supported source files: `.md`, `.markdown`, `.mdx`, `.txt`
2. analyze each source file path, title, headings, and hint
3. choose an existing adapter leaf spec when possible
4. append a converted import block into that target
5. preserve the full original source file in the target
6. refresh `.superpowers/spec` indexes

If the whole directory should be biased toward a topic:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/spec_import.py path/to/original-spec-dir --hint "api contract"
```

If all source specs should be placed under a target adapter directory:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/spec_import.py path/to/original-spec-dir --target imported
```

If a single source file should go to a specific adapter leaf:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/spec_import.py path/to/source-spec.md --target imported/source-spec.md
```

## Existing `.superpowers/spec` merge behavior

If `.superpowers/spec` already has files, import must merge into them.

The script does this by:

- selecting existing leaf specs through the same selector used by update-spec
- appending a new imported block instead of replacing file content
- adding an import source marker with a content digest
- skipping an import when the exact same source file content has already been imported

## Expected imported block shape

Each imported source file becomes a block like this inside an adapter leaf spec:

```markdown
<!-- superpower-adapter:import-source:<digest>:<source-relative-path> -->
## Update: Imported Spec - <Imported Title>

### Why
- ...

### Rules / Contracts
- ...

### Validation / Notes
- ...

## Imported Original Spec

<!-- superpower-adapter:imported-original:start -->
```markdown
<original source spec, verbatim>
```
<!-- superpower-adapter:imported-original:end -->
```

## Completion checklist

Before finishing:

- [ ] The source directory was scanned recursively.
- [ ] All source files intended for import were converted or explicitly skipped as already imported.
- [ ] Existing `.superpowers/spec` content was preserved.
- [ ] Each converted section is adapter-style structured markdown.
- [ ] Every source file's full content is preserved verbatim in an original-content archive.
- [ ] `update-spec.py` refreshed the index chain.
- [ ] You told the user which source files mapped to which adapter target files.

## Relationship to update-spec

Use `import-spec` for one-time conversion of an existing user spec directory into adapter format.
Use `update-spec` later for ongoing durable knowledge updates discovered during implementation, debugging, review, or discussion.

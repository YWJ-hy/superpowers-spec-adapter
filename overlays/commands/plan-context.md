Generated by superpower-adapter.

Use this command when a plan under `docs/superpowers/plans/*.md` needs a stable sidecar context bundle.

## Common commands

Initialize the sidecar and set the current plan pointer:

```bash
python3 superpowers/scripts/plan-context.py init docs/superpowers/plans/<stem>.md --set-current
```

Show the current plan and sidecar path:

```bash
python3 superpowers/scripts/plan-context.py current
```

Add planning-selected spec context:

```bash
python3 superpowers/scripts/plan-context.py add --phase plan --spec .superpowers/spec/topic/example.md --reason "Explains the contract this plan changes"
```

Add implementation-only context:

```bash
python3 superpowers/scripts/plan-context.py add --phase implement --spec .superpowers/spec/guides/testing.md --reason "Implementation needs the testing checklist"
```

Add review-only context:

```bash
python3 superpowers/scripts/plan-context.py add --phase review --spec .superpowers/spec/guides/review.md --reason "Review should verify this checklist"
```

Render implementation context:

```bash
python3 superpowers/scripts/plan-context.py render --phase implement
```

Render review context:

```bash
python3 superpowers/scripts/plan-context.py render --phase review
```

Verify the current plan sidecar:

```bash
python3 superpowers/scripts/plan-context.py verify --current
```

## Usage notes

- Keep the primary plan file at `docs/superpowers/plans/<stem>.md`.
- Store sidecar files at `docs/superpowers/plans/<stem>.context/`.
- Keep `.superpowers/current-plan` as a lightweight pointer to the active plan.
- Prefer summary-mode records unless full text is truly needed.

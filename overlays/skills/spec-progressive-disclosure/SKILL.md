Generated by superpower-adapter.

This skill is a reference and fallback guide, not the default spec selection path.

Default adapter flow:
1. During `brainstorming`, use `spec-researcher` with `phase: brainstorm` to disclose a small amount of relevant project spec context.
2. During `writing-plans`, use `spec-researcher` with `phase: plan` to formally select relevant project specs.
3. Record selected specs and constraints in the plan's `## Referenced Project Specs` section.
4. During implementation and review, consume `Referenced Project Specs` instead of reselecting specs from scratch.
5. After finishing work, use the `update-spec` skill to review whether durable implementation knowledge should persist in `.superpowers/spec/`.

Use this skill only when:
- the user asks how progressive spec disclosure works;
- `spec-researcher` is unavailable and a manual fallback is needed;
- you are debugging adapter installation or native skill patch behavior.

Reference rules:
1. Start with `.superpowers/spec/index.md`.
2. Follow links or listed paths to narrower indexes before reading leaf specs.
3. Read only the minimum detailed spec files needed for the current phase.
4. Do not read the whole `.superpowers/spec/` tree unless the human explicitly asks for a full audit.
5. If `.superpowers/spec/index.md` is missing, report that project specs are unavailable instead of guessing paths.
6. Treat adapter sidecar JSONL files as diagnostics only, not as the required source of truth for new work.

Manual fallback flow:
- Understand task
- Read `.superpowers/spec/index.md`
- Follow links or listed paths to narrower indexes
- Read the minimum relevant leaf spec files
- If planning, write selected specs into `Referenced Project Specs`
- If implementing or reviewing, use the plan's `Referenced Project Specs`
- After finishing work, update relevant spec documents if new knowledge should persist

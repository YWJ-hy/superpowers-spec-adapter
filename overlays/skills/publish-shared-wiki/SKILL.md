Use this skill when the user wants to publish shared wiki updates that live in `.shared-superpowers/wiki/` as a git submodule or git repository managed through project-local `.shared-superpowers/settings.json` hooks.

This skill is for publishing shared wiki submodule changes only. It does not replace `update-wiki` review for durable implementation knowledge.

If the user is using the GitHub-backed shared-wiki MCP flow, do not use this skill for that change. Use the `shared-wiki-mcp` skill or the `update-wiki` MCP path to validate a patch and open a branch + PR instead.

## Goal

Confirm the shared wiki submodule is ready, ask the user to confirm the commit/push scope, then publish the submodule changes and update the parent repository pointer.

## Non-negotiable rules

- Treat this as a standalone adapter skill, not as a Superpowers development workflow step.
- Do not invoke Superpowers planning, implementation, review, completion, or verification skills before or after this skill.
- Do not commit or push without explicit user confirmation of the computed commit/push scope immediately before publishing.
- Do not overwrite unrelated parent repository changes.
- Do not initialize the submodule here; this skill only publishes an existing shared wiki submodule/repository.
- Do not publish shared wiki content that contains system-specific identifiers, internal URLs, environment names, local paths, or current-system-only business rules.
- Do not bypass `update-wiki`; durable implementation knowledge still belongs there.

## Workflow

1. Confirm the project root and the shared wiki path.
2. Read `.shared-superpowers/settings.json` if needed to confirm the configured hook names.
3. Run the verify hook first:

```bash
python3 ./.shared-superpowers/scripts/run-hook.py sharedWikiSubmodule:verify
```

4. Run the shared wiki mechanical validator, including configured neutrality guards:

```bash
python3 __SUPERPOWER_ADAPTER_PLUGIN_ROOT__/scripts/wiki_update_check.py --wiki-root shared
```

5. Tell the user exactly what will be committed and pushed, including the shared wiki repository/submodule state and the parent repository pointer change.
6. Ask the user for explicit confirmation of that computed commit/push scope. Do not treat earlier broad instructions such as "publish" as sufficient authorization for this step.
7. Run the publish hook only after that confirmation:

```bash
python3 ./.shared-superpowers/scripts/run-hook.py sharedWikiSubmodule:publish
```

8. Run the status hook to confirm the final state:

```bash
python3 ./.shared-superpowers/scripts/run-hook.py sharedWikiSubmodule:status
```

9. Report the final parent pointer state and any remaining caveats.
